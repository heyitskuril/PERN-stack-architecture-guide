# Contributing

Contributions to this guide are welcome. The goal is to keep the content accurate, current, and grounded in real engineering practice — not opinion alone.

## What Makes a Good Contribution

A good contribution to this guide is one that:

- Corrects a factual error (broken link, outdated recommendation, incorrect code)
- Adds a reference that is missing but genuinely relevant
- Improves an explanation that is unclear or incomplete
- Adds a section that fits the scope of the guide and is backed by credible sources

A contribution is not a good fit if it:

- Adds opinion without a cited reference
- Promotes a specific tool or library without addressing trade-offs
- Rewrites existing content without a clear reason

## How to Contribute

1. Open an issue first for any significant change. This allows discussion before you spend time writing.
2. Fork the repository and create a branch from `main`:
   ```bash
   git checkout -b fix/broken-prisma-link
   git checkout -b feat/add-redis-caching-section
   ```
3. Make your changes.
4. Open a pull request with a clear description of what changed and why.

## Submitting a Pull Request

- Keep pull requests focused on one change at a time
- Reference the issue number if one exists
- Update the relevant section's sources list if you are adding a reference

## Reporting Issues

Use the issue templates for:
- Broken links or outdated content
- Requests to add a missing resource or section

For anything else, open a blank issue with a clear description.
